# Decision log

Design rulings made while building this kit, and the roadmap items deferred out
of them. One line of rationale each; the code is the full story.

## Decisions

| Decision | Date | Rationale |
|---|---|---|
| Config over code: engine prompt sections (SPEAKING RULES, STYLE, GOAL, FLOW, QUALIFY, IDENTITY body, medical block) are engine-owned and **not** client-configurable | 2026-07-18 | The differences between clients are narrow (facts, voice, greeting, guardrails); making the shared craft configurable would recreate the bespoke-per-client trap the kit exists to avoid. |
| Deterministic pipeline; the LLM owns dialogue only | 2026-07-18 | Deploy, diff, gating, SMS, and evals are plain Python so they're testable and idempotent; the model is confined to the live conversation where its strengths are. |
| Lockfile idempotency + dry-run as the default | 2026-07-18 | `config/clients/<slug>.lock.json` maps language→{agent_id,llm_id}; plans diff desired vs live and re-runs converge to NOOP, so applying twice is safe and the default command mutates nothing. |
| Voice IDs are config, not secrets | 2026-07-18 | A Retell `voice_id` (stock or account custom) is not sensitive; it belongs in the client config next to the greeting, not in `.env`. |
| post_call caller/derived split + explicit role markers | 2026-07-18 | `source: caller\|derived` decides what CAPTURING DETAILS asks for vs. what's summarized after; `role: phone\|consent` resolves the SMS fields deterministically instead of by name heuristics (heuristics remain as a documented fallback). |
| sms_consent requires the whole pipeline | 2026-07-18 | A booking SMS only sends when sms_consent + booking.url + captured consent + a normalizable phone all hold, the service is running, and Twilio is configured; the deploy warns when consent is on but no SMS backend exists. |
| DTMF: prompt follows config | 2026-07-18 | Production keeps `allow_user_dtmf = True`; rather than fight it, the SPEAKING RULES text tells the agent keyed digits may arrive as input and to read them back — config and prompt agree. |
| Sanitized-public / private-config split | 2026-07-18 | The tracked `client.example.yaml` is obviously fictional; real client configs, lockfiles, capture dumps, and the SMS ledger are all gitignored so nothing private is ever committed. |
| Static evals gate CI; transcript evals run out-of-band | 2026-07-19 | Layer 1 needs no network and runs on every push; Layer 2 needs real transcripts, so it stays a local/manual tool until sanitized fixtures exist. |
| Retell list endpoints: verified none deprecated; hardened to the paginated contract | 2026-07-22 | The June 2026 legacy-list deprecation notice covers `/v2/list-calls` (→ `/v3/list-calls`) and several unversioned GET list endpoints, but **not** `list-agents`. We already use `/v2/list-agents` (current) and `/v3/list-calls` (the replacement), so no endpoint change was needed. Both list callers now page the documented `{items, pagination_key, has_more}` envelope instead of reading a single page. |
| Eval v2 initiated: run manifests + prompt fingerprinting as the regression-tracking spine; transcript runner advisory by default with an explicit `--strict` opt-in | 2026-07-23 | Each eval run now writes a JSON manifest (`var/evals/runs/<run_id>.json`) pinning git commit, per-`slug/language` sha256 prompt fingerprints, sources, and findings; the byte-deterministic compiler makes each fingerprint a stable prompt-version ID, so regressions are diffable across runs. The transcript runner stays advisory (always exit 0) unless `--strict` is passed. The previously-deferred fixture-runner ruling (static evals gate CI; transcript evals run out-of-band) will be revisited in an upcoming step: CI-gating moves to synthetic fixtures while real calls stay advisory. |
| LLM-judge un-deferred and shipped as Layer 4 (advisory) | 2026-07-23 | `evals/judge_checks.py` + `evals/run_judge.py` judge each transcript against a fixed rubric (booking intent, hallucinated commitment vs. the compiled FACTS, unresolved request) via the Anthropic Messages API over raw httpx (no SDK, no new dependency; model `claude-sonnet-4-6`). The deterministic layers (static/fixtures) gate CI; the judge only advises — it needs an API key and CI has none, so `ci.yml` is untouched. **Citation enforcement is the honesty mechanism**: a "fail" whose quote is not found verbatim in a cited turn is downgraded to a `judge_citation_unverified` finding rather than asserted; a malformed/unknown-shape reply becomes `judge_output_invalid`, never a crash. `--max-calls` caps API-judged calls per run for cost; `--dry-run` gives a keyless zero-network smoke test. This resolves the 2026-07-20 roadmap item (then numbered Layer 3, renumbered to Layer 4 once latency shipped). |
| Regression compare CLI added as a pure reader over run records | 2026-07-23 | `evals/compare_runs.py` diffs two same-layer records (explicit paths or `--layer <L> --latest`) and defines a regression as **any new finding in the candidate vs the baseline** (exit 1); resolved/persisting findings and prompt-fingerprint drift are reported but never fail on their own — a prompt may change freely as long as no check newly fires. Cross-layer comparison and `<2` records are hard errors (exit 2); differing source sets emit an honest NOTE that the finding delta may reflect data, not prompts. It reads records only — no re-running checks, no network. |
| Golden fixture suite added and gates CI (eval Layers 2+3 over synthetic calls) | 2026-07-23 | `evals/fixtures/*.json` are fully synthetic Acme-Wellness calls, each pinning the exact transcript/latency check names it must produce; `evals/run_fixtures.py` replays the checks against `client.example.yaml` and exits nonzero on any mismatch, malformed fixture, or typo'd (unknown) expected check name. This formally reverses the 2026-07-19 "transcript evals run out-of-band" ruling **for synthetic coverage only** — real-call runs (`run_transcripts.py` / `run_latency.py`) stay advisory/out-of-band. Rationale: once we claim regression detection, a CI-runnable benchmark is the product, so discrimination power is proven continuously rather than by one-off tests. |
| Latency budgets added as a third advisory eval layer; fingerprint keys switched to source-path-qualified form | 2026-07-23 | `evals/latency_checks.py` + `evals/run_latency.py` compare Retell per-call `e2e`/`llm`/`tts`/`asr` percentiles against a documented `LatencyBudget` (initial operating targets grounded in voice-UX norms and observed production behavior — tunable engine defaults, not a vendor SLA), writing records under the new `latency` layer; advisory by default with `--strict`, missing data surfaced as findings rather than skipped. Separately, `prompt_fingerprints` keys moved from `slug/code` to `path::code` (`fingerprint_key`) to eliminate the same-slug collision (example + local `acme-wellness.yaml`) before regression tooling depends on the keys; record shape unchanged, `schema_version` stays 1. |

## Roadmap (deferred, not built)

| Item | Date recorded | Rationale for deferral |
|---|---|---|
| LLM-judge eval layer (Layer 4) | 2026-07-20 | Deterministic checks land first; a judged layer for tone/faithfulness is additive and needs a rubric + cost budget. (Renumbered from Layer 3 once latency budgets shipped as Layer 3.) |
| Voice pre-flight inside `plan` | 2026-07-20 | We caught an invented voice id by hand; the planner should validate every `voice_id` against `list-voices` before apply. |
| Incremental per-language lockfile writes | 2026-07-20 | `apply` writes the lockfile once at the end; a mid-apply failure can orphan resources. Write per language so partial applies self-heal. |
| Field ownership / explicit unset | 2026-07-20 | The differ only sees emitted fields; omitted ones (e.g. `webhook_url` when unset) can't be detected or cleared through a plan. |
| Delete / deprovision support | 2026-07-20 | There is no `--delete`; tearing a client down is a manual API call. A first-class deprovision (with lockfile cleanup) is needed. |
| Hours "closed day" model | 2026-07-20 | `Hours` is `{days, open, close}`; a closed day renders as "Closed to Closed". A first-class closed concept would read better. |
| Per-client SMS numbers | 2026-07-20 | One global `TWILIO_FROM_NUMBER` today; multi-client at scale needs a per-client sender and A2P/toll-free registration per number. |
| Named tunnel / VPS deployment for the webhook service | 2026-07-20 | The quick tunnel is ephemeral; production post-call handling needs a stable, always-on host. |
